## Overview

SpaceFlapper supports runtime language switching across 5 languages without requiring an app restart. The `LocalizationManager` singleton loads language-specific `.lproj` bundles, persists the user's preference in `UserDefaults`, and publishes changes so SwiftUI views automatically refresh.

## Supported languages

| Code | Language | Native Name | Flag |
|------|----------|-------------|------|
| `en` | English | English | GB |
| `es` | Spanish | Espanol | ES |
| `fr` | French | Francais | FR |
| `de` | German | Deutsch | DE |
| `ja` | Japanese | Japanese | JP |

## Architecture

```mermaid

    A[SpaceFlapperApp] -->|.environmentObject| LM[LocalizationManager.shared]

    V -->|L key| GF[Global L Function]

```

### Key components

| Component | Role |
|-----------|------|
| `LocalizationManager` | Singleton `ObservableObject` that manages language state |
| `.lproj` bundles | Standard iOS localization bundles under `Resources/` |
| `L()` function | Global convenience function for string lookup |
| `UserDefaults` | Stores the user's language preference |

## LocalizationManager

The `LocalizationManager` class is a singleton `ObservableObject` that SwiftUI views observe for language changes:

```swift LocalizationManager.swift
final class LocalizationManager: ObservableObject {
    static let shared = LocalizationManager()

    @Published var currentLanguage: String {
        didSet {
            if oldValue != currentLanguage {
                loadBundle(for: currentLanguage)
            }
        }
    }

    static let availableLanguages = ["en", "es", "fr", "de", "ja"]
    private(set) var bundle: Bundle = .main
}
```

<Callout kind="info">
  The `@Published` property wrapper on `currentLanguage` triggers SwiftUI view updates whenever the language changes at runtime.
</Callout>

### Language detection on first launch

When no saved preference exists, the manager detects the device language and falls back to English:

```swift LocalizationManager.swift
private init() {
    if let saved = UserDefaults.standard.string(forKey: userDefaultsKey),
       Self.availableLanguages.contains(saved) {
        self.currentLanguage = saved
    } else {
        let preferred = Locale.preferredLanguages.first?
            .prefix(2).lowercased() ?? "en"
        self.currentLanguage = Self.availableLanguages
            .contains(String(preferred)) ? String(preferred) : "en"
    }
    loadBundle(for: currentLanguage)
}
```

### Setting a language

The `setLanguage` method persists the choice and updates the active bundle:

```swift LocalizationManager.swift
func setLanguage(_ code: String) {
    guard Self.availableLanguages.contains(code) else { return }
    UserDefaults.standard.set(code, forKey: userDefaultsKey)
    UserDefaults.standard.set([code], forKey: "AppleLanguages")
    currentLanguage = code
}
```

Setting `AppleLanguages` ensures iOS system-level formatters (dates, numbers) also use the selected locale.

### Bundle loading

When the language changes, `loadBundle` locates the corresponding `.lproj` directory:

```swift LocalizationManager.swift
private func loadBundle(for languageCode: String) {
    if let path = Bundle.main.path(forResource: languageCode, ofType: "lproj"),
       let languageBundle = Bundle(path: path) {
        bundle = languageBundle
    } else {
        bundle = .main  // Fallback to English
    }
}
```

## The global L() function

A global convenience function provides concise string lookup anywhere in the codebase:

```swift LocalizationManager.swift
func L(_ key: String, comment: String = "") -> String {
    return LocalizationManager.shared.localizedString(key, comment: comment)
}
```

Usage throughout the app:

```swift
Text(L("menu.tap_to_start"))
Text(String(format: L("game_over.total"), stardustAmount))
```

## String key patterns

Localization keys follow a consistent dot-notation pattern organized by feature:

| Pattern | Example | Usage |
|---------|---------|-------|
| `menu.*` | `menu.tap_to_start` | Main menu UI |
| `game_over.*` | `game_over.score` | Game over screen |
| `shop.*` | `shop.equipped` | Suit shop |
| `achievements.*` | `achievements.unlocked` | Achievement display |
| `statistics.*` | `statistics.best_score` | Statistics screen |
| `settings.*` | `settings.language` | Settings screen |
| `suit.[id].name` | `suit.neon.name` | Suit names |
| `suit.[id].description` | `suit.cosmic.description` | Suit descriptions |
| `achievement.[id].name` | `achievement.first_flight.name` | Achievement names |
| `achievement.[id].description` | `achievement.space_ace.description` | Achievement descriptions |
| `difficulty.*` | `difficulty.hard` | Difficulty tier names |
| `powerup.*` | `powerup.star_shield` | Power-up names |
| `badge.*` | `badge.near_miss` | Run recap badges |
| `micro_goal.*` | `micro_goal.streak` | Micro-goal suggestions |

<Callout kind="tip">
  Format strings use `%d` for integers and `%@` for strings, following Apple's `NSLocalizedString` conventions. For example: `"Just %d away!"`.
</Callout>

## App entry point integration

`SpaceFlapperApp` injects the `LocalizationManager` as an `@EnvironmentObject` so all views in the hierarchy can observe language changes:

```swift SpaceFlapperApp.swift
@main
struct SpaceFlapperApp: App {
    @ObservedObject private var localizationManager = LocalizationManager.shared

    var body: some Scene {
        WindowGroup {
            ContentView()
                .environmentObject(localizationManager)
        }
    }
}
```

## Adding a new language

<Steps>
  <Step title="Create the .lproj bundle" icon="folder-plus" titleType="p">
    Add a new directory under `Resources/` named with the language code (e.g., `pt.lproj/`) and create a `Localizable.strings` file inside.
  </Step>

  <Step title="Translate all keys" icon="languages" titleType="p">
    Copy `en.lproj/Localizable.strings` and translate every value. Preserve the keys and format specifiers (`%d`, `%@`).
  </Step>

  <Step title="Register the language" icon="settings" titleType="p">
    Add the language code to the `availableLanguages` array and `languageDisplayNames` / `languageFlags` dictionaries in `LocalizationManager`:

    ```swift
    static let availableLanguages = ["en", "es", "fr", "de", "ja", "pt"]

    static let languageDisplayNames: [String: String] = [
        // ... existing entries
        "pt": "Portugues"
    ]

    static let languageFlags: [String: String] = [
        // ... existing entries
        "pt": "flag-emoji"
    ]
    ```
  </Step>

  <Step title="Add the settings row" icon="list" titleType="p">
    Add a new case in `SettingsView.languageRow` for the language name key:

    ```swift
    case "pt": return "settings.language.portuguese"
    ```

    Then add the corresponding key to every existing `Localizable.strings` file.
  </Step>
</Steps>

## Locale-aware formatting

Views that display numbers or time use the current language's locale for formatting:

```swift StatisticsView.swift
private var selectedLocale: Locale {
    let identifier = localizationManager.bundle
        .preferredLocalizations.first ?? localizationManager.currentLanguage
    return Locale(identifier: identifier)
}

private func formatNumber(_ value: Int) -> String {
    let formatter = NumberFormatter()
    formatter.numberStyle = .decimal
    formatter.locale = selectedLocale
    return formatter.string(from: NSNumber(value: value)) ?? "\(value)"
}
```

This ensures number separators (e.g., 1,000 vs 1.000) match the selected language's conventions.
