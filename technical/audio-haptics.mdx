## Overview

SpaceFlapper uses programmatically generated sound effects via `AVAudioEngine` and Core Haptics feedback through `CHHapticEngine`. Both systems are singletons, fire-and-forget, and respect the shared audio/haptic toggle controlled by `GameManager.isAudioEnabled`.

## SoundManager

### Architecture

`SoundManager` is a singleton (`SoundManager.shared`) built on `AVAudioEngine`. All sound effects are synthesized programmatically using `AVAudioPCMBuffer` - no audio asset files are used. Buffers are pre-generated and cached for zero-latency playback.

```swift SoundManager.swift
final class SoundManager {
    static let shared = SoundManager()
    private var audioEngine: AVAudioEngine?
    private var playerNode: AVAudioPlayerNode?
    private var musicPlayerNode: AVAudioPlayerNode?
}
```

## HapticManager

### Architecture

`HapticManager` is a singleton (`HapticManager.shared`) wrapping `CHHapticEngine` from Core Haptics. It gracefully no-ops on devices without haptic hardware (simulator, older iPads). The engine lazily starts on first use and auto-restarts after interruptions.

```swift HapticManager.swift
final class HapticManager {
    static let shared = HapticManager()
    private var engine: CHHapticEngine?
    private var supportsHaptics: Bool

    private init() {
        supportsHaptics = CHHapticEngine.capabilitiesForHardware().supportsHaptics
    }
}
```

### Engine lifecycle

```swift HapticManager.swift
private func ensureEngine() {
    guard supportsHaptics, engine == nil else { return }
    let eng = try CHHapticEngine()

    eng.resetHandler = { [weak self] in
        try? self?.engine?.start()
    }
    eng.stoppedHandler = { [weak self] reason in
        self?.engine = nil
    }

    try eng.start()
    engine = eng
}
```

<Callout kind="tip">
  The `resetHandler` automatically restarts the engine when returning from background. The `stoppedHandler` nils the engine reference so it gets re-created on next use. All haptic calls use fire-and-forget error handling to never block the game loop.
</Callout>

### Multi-event patterns

#### Streak level-up (double-tap)

```swift HapticManager.swift
func playStreakLevelUp() {
    let tap1 = CHHapticEvent(eventType: .hapticTransient,
        parameters: [intensity: 0.6, sharpness: 0.6], relativeTime: 0)
    let tap2 = CHHapticEvent(eventType: .hapticTransient,
        parameters: [intensity: 0.6, sharpness: 0.6], relativeTime: 0.1)
    playPattern(events: [tap1, tap2])
}
```

### Speed surge continuous vibration

The speed surge event uses a continuous haptic pattern that runs for the entire surge duration:

```swift HapticManager.swift
func startSpeedSurge() {
    let event = CHHapticEvent(
        eventType: .hapticContinuous,
        parameters: [intensity: 0.2, sharpness: 0.3],
        relativeTime: 0,
        duration: 30  // Long duration, stopped manually
    )
    let player = try engine.makeAdvancedPlayer(with: pattern)
    try player.start(atTime: CHHapticTimeImmediate)
    speedSurgePlayer = player
}

func stopSpeedSurge() {
    try? speedSurgePlayer?.stop(atTime: CHHapticTimeImmediate)
    speedSurgePlayer = nil
}
```

## Audio/haptic toggle

Both managers share the same toggle controlled by `GameManager.isAudioEnabled`:

```swift GameManager.swift
@Published var isAudioEnabled: Bool = true {
    didSet {
        SoundManager.shared.setAudioEnabled(isAudioEnabled)
        HapticManager.shared.setEnabled(isAudioEnabled)
        saveAudioPreference()
    }
}
```

The preference is persisted to `UserDefaults` under the key `SpaceFlapper.AudioEnabled`.
