## Organization principles

### Separation by concern

| Directory | Responsibility | Key classes |
|-----------|---------------|-------------|
| `Game/` | All SpriteKit gameplay logic | `GameScene`, `PlayerNode`, `ObstacleManager` |
| `Views/` | SwiftUI overlay screens | `MenuOverlayView`, `PlayingOverlayView` |
| `Progression/` | Persistent player data | `ProgressionManager`, `PlayerProgress` |
| `Managers/` | App-wide services | `LocalizationManager` |
| `Resources/` | Static assets | Localization strings, audio files |

### Naming conventions

SpaceFlapper follows consistent naming patterns throughout the codebase:

- **Manager suffix** -- Classes that coordinate subsystems: `ObstacleManager`, `DifficultyManager`, `ComboManager`
- **Node suffix** -- SpriteKit node subclasses: `PlayerNode`, `ObstacleNode`, `MeteorNode`, `PowerUpNode`
- **Obstacle suffix** -- Specialized obstacle types: `GravityWellObstacle`, `SolarFlareObstacle`
- **View suffix** -- SwiftUI views: `MenuOverlayView`, `ShopView`, `SettingsView`
- **PowerUp suffix** -- Power-up variants: `StarShieldPowerUp`, `RocketBoostPowerUp`

<Callout kind="tip">
  The `Game/` directory contains both the core engine and all obstacle/event types. If the project grows further, obstacle types and event managers could be extracted into subdirectories for clearer separation.
</Callout>

## Zero external dependencies

SpaceFlapper uses only Apple-provided frameworks:

- **SwiftUI** -- User interface layer and state management
- **SpriteKit** -- 2D game rendering, physics simulation, particle effects
- **GameplayKit** -- State machine for game flow (`GKStateMachine`, `GKState`)
- **Combine** -- Reactive data binding via `@Published` and `ObservableObject`
- **UIKit** -- Procedural texture generation and haptic feedback
- **Foundation** -- `UserDefaults` persistence, `JSONEncoder`/`JSONDecoder`

<Callout kind="info">
  Every visual asset in the game -- the astronaut sprite, obstacles, backgrounds, and particle effects -- is generated procedurally in code. There are no image assets in the game bundle beyond the app icon.
</Callout>
